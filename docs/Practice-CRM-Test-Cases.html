1. Scope & Conventions

Application at deterministic seed state (open with /?reset=true). Password for all users: Pass@123. Default login: admin@crm.com unless stated. Seed: 50 leads, 40 contacts, 20 accounts, 12 products, 25 deals, 30 tasks, 15 tickets, 5 users. All data operations have simulated latency (300–1200 ms) — wait for spinners/skeletons to resolve before asserting.

  • Tags: Smoke critical path, every build · Sanity post-deploy health (includes Smoke) · Regression full coverage (includes everything) · Feature one feature exercised exhaustively (happy path, negatives, edges, persistence) · E2E start-to-finish journeys, nightly.
  • Every test case is independent; run /?reset=true before a case (or suite) that needs pristine data.
  • Locators: element ids (el-xxxxx-n) are decoys that change every page load — never use them. Prefer roles, visible text, placeholders, and documented data-testid attributes (see LOCATORS.md).
  • After any action that saves or navigates, wait for destination evidence (URL, page container, expected text) before asserting — latency is randomized by design.
  • Demo users: admin@crm.com (admin), rep@crm.com (rep), viewer@crm.com (viewer), priya@crm.com (deactivated). Forgot-password code: 123456.
